## 🐛 Common Errors & Solutions

### Error: "Location not detected"
```
Solution:
1. Check browser permissions
2. Use localhost (not 192.168.x.x)
3. Try HTTPS if deployed
4. Restart browser
```

### Error: "No places found"
```
Reasons:
- Remote location with few places
- API key not configured
- Search radius too small

Solutions:
- Try broader search: "restaurants" instead of "Italian restaurants"
- Check .env file for GOOGLE_PLACES_API_KEY
- System will show dummy data if API unavailable
```

### Error: "Booking link not working"
```
Note: Booking links open external sites
- Make sure pop-ups allowed
- Check internet connection
- Try different browser
```

## 📊 API Usage Tracking

To monitor your usage:

```python
# In terminal, you'll see:
[CACHE HIT] Returning cached results for restaurant
[API SUCCESS] Found 6 restaurants
[REALTIME GUIDE] Searching for hotel near 26.9124, 75.7873
```

### Cache Indicators
- `[CACHE HIT]` = No API call (saved money!)
- `[API SUCCESS]` = New API call
- `[EXCEPTION]` = Error, using dummy data
